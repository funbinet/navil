# Navil — Autonomous Self-Learning Bug Hunter

> An AI-powered offensive security platform that scans web applications, learns vulnerability patterns through reinforcement learning, dynamically mutates payloads, and evolves its attack strategies from successful exploits.

## Background & Problem Statement

Traditional bug bounty and security scanning tools are **static and rule-based** — they run the same payloads against the same patterns every time. They don't learn from past findings, they don't adapt to WAF evasion, and they can't chain vulnerabilities into complex attack paths. Navil changes this by introducing a **self-evolving offensive security system** built on reinforcement learning, where every scan makes the system smarter.

> [!IMPORTANT]
> **Legal & Ethical Disclaimer**: Navil is designed **exclusively for authorized security testing**. It must only be used against systems you have explicit, written permission to test. The system enforces scope boundaries, requires target authorization tokens, and logs all actions for audit compliance. Unauthorized use is illegal and unethical.

## User Review Required

> [!IMPORTANT]
> ### Scope & Authorization Model
> Navil enforces a strict **scope file** (`.navil-scope.yml`) that defines allowed domains, paths, and methods. The scanner will **refuse to operate** without a valid scope file. Should we also require a cryptographic authorization token signed by the target owner?

> [!WARNING]
> ### Reinforcement Learning Training
> The RL agent needs training data. The initial approach uses **offline pre-training** on curated vulnerability datasets (OWASP WebGoat, DVWA patterns) before live scanning. Training on real targets requires careful rate-limiting to avoid causing disruption. Are you comfortable with the RL agent starting with pre-trained weights and fine-tuning during authorized scans?

> [!IMPORTANT]
> ### Dashboard Deployment
> The dashboard will be served by FastAPI as static files with WebSocket for real-time updates. This keeps the architecture simple (no separate frontend build). Is this acceptable, or do you prefer a React/Vue frontend?

### 3. Vulnerability Scanner (`navil/scanner/`)

Plugin-based vulnerability detection with 12 built-in detectors.

#### [NEW] [base.py](file:///home/bane/navil/navil/scanner/base.py)
```python
class VulnPlugin(ABC):
    """Abstract base for all vulnerability detection plugins."""
    name: str
    severity: Severity  # CRITICAL, HIGH, MEDIUM, LOW, INFO
    category: str       # OWASP category
    
    @abstractmethod
    async def scan(self, target: Target, context: ScanContext) -> list[Finding]: ...
    
    @abstractmethod
    def get_payloads(self) -> list[Payload]: ...
    
    @abstractmethod
    def verify(self, response: Response, payload: Payload) -> bool: ...
```

### 4. Reinforcement Learning Brain (`navil/brain/`)

The self-learning core that makes Navil intelligent.

#### [NEW] [environment.py](file:///home/bane/navil/navil/brain/environment.py)
- Custom Gymnasium environment for web vulnerability scanning
- **State Space**: Vector encoding of target properties (tech stack, input count, response codes, WAF detected, scan history)
- **Action Space**: `{choose_plugin, choose_payload_strategy, set_encoding, adjust_rate, chain_previous_finding}`
- **Reward Function**:
  - `+10.0` — New unique vulnerability confirmed
  - `+5.0` — New unique suspicious behavior
  - `+3.0` — New endpoint discovered
  - `+1.0` — Successful payload that bypassed WAF
  - `-0.5` — Duplicate finding
  - `-1.0` — Target timeout/error caused
  - `-5.0` — Out-of-scope attempt (hard penalty)

#### [NEW] [agent.py](file:///home/bane/navil/navil/brain/agent.py)
- Stable-Baselines3 DQN agent with custom network
- Double DQN with prioritized experience replay
- Epsilon-greedy exploration with decay
- Model save/load for transfer learning
- Online learning during scans (fine-tuning)

#### [NEW] [trainer.py](file:///home/bane/navil/navil/brain/trainer.py)
- Offline pre-training on curated vulnerability datasets
- Online training during authorized scans
- Periodic model checkpointing
- Training metrics logging (TensorBoard compatible)
- Curriculum learning: easy targets → hard targets

#### [NEW] [memory.py](file:///home/bane/navil/navil/brain/memory.py)

## Feedback Loop Architecture

This is the core innovation — the self-learning cycle:

```mermaid
flowchart LR
    A[🔍 Scan Target] --> B[📊 Collect Results]
    B --> C{Finding?}
    C -->|Yes| D[✅ Positive Reward]
    C -->|No| E[❌ Negative Reward]
    D --> F[🧬 Extract Pattern]
    E --> G[🔄 Mutate Strategy]
```

### How the Feedback Loop Works:

1. **Scan** → Crawler discovers endpoints, scanner tests them
2. **Result** → Finding or no finding
3. **Reward** → RL agent receives positive/negative reward
4. **Learn** → Agent updates its Q-table/policy network
5. **Adapt** → Next scan uses improved strategy:
   - Which plugins to try first
   - Which payload encodings work best
   - What parameter patterns are most vulnerable
   - How to evade detected WAFs
6. **Mutate** → Genetic algorithm evolves payloads based on what worked
7. **Repeat** → Each scan makes the system smarter
